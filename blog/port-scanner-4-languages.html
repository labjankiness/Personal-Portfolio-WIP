Writing a Port Scanner in 4 Languages

March 2026

The idea

Port scanning is one of the first things you learn in cybersecurity. Tools like Nmap do it well, but I wanted to understand what's actually happening at the socket level. So I built the same TCP port scanner in four languages: Python, C, Go, and Rust.

The goal wasn't just to get it working — it was to compare how each language handles concurrency, error handling, and raw network I/O.

Python: the quick prototype

Python was the fastest to write. Using socket and concurrent.futures.ThreadPoolExecutor, I had a working multi-threaded scanner in under 50 lines. It connects to each port with a timeout, grabs the service banner if available, and prints results.

The downside? Speed. Even with threading, Python's GIL means you're not truly parallel. For scanning 65,535 ports, it's noticeably slower than the compiled alternatives.

C: close to the metal

The C version uses POSIX sockets and pthreads. It's more code — you manually manage memory, handle errno, and build your own thread pool. But it's fast. Really fast.

Writing it in C gave me a much deeper understanding of what connect() actually does at the syscall level, and how select() or poll() can be used for non-blocking I/O.

Go: concurrency done right

Go was the sweet spot. Goroutines make concurrent scanning trivial — just go scanPort(port) inside a loop with a semaphore channel to limit concurrency. The net.DialTimeout function handles connection attempts cleanly.

The resulting code is almost as short as Python but runs nearly as fast as C. Go's concurrency model is genuinely impressive for network tools.

Rust: safety and speed

Rust was the hardest to write but produced the most robust code. The borrow checker caught several bugs at compile time that would have been runtime issues in C. Using tokio for async I/O and TcpStream::connect_timeout, the scanner is both fast and memory-safe.

Error handling with Result<T, E> forced me to handle every possible failure case, which is exactly what you want in a security tool.

Performance comparison

Scanning the first 1024 ports on localhost (approximate results):

Language Time Lines of Code Concurrency Model
Python ~12s ~45 ThreadPoolExecutor
C ~2s ~150 pthreads
Go ~3s ~60 goroutines + channels
Rust ~2s ~80 tokio async

Key takeaways

  • Python is great for prototyping security tools but hits a wall with performance-sensitive tasks.
  • C gives you full control but demands careful memory management. One wrong pointer and your scanner becomes a vulnerability.
  • Go is the pragmatic choice for network tools. Goroutines are simple, fast, and hard to misuse.
  • Rust is the best choice when correctness matters. The compiler catches bugs that would slip through in every other language.

If I were building a production security tool, I'd pick Go for speed of development or Rust for maximum reliability. For a quick audit script, Python still wins.
