OverTheWire Bandit: A Beginner's CTF Walkthrough

March 2026

What is OverTheWire Bandit?

Bandit is the entry-level wargame on OverTheWire. It teaches the basics of Linux command-line skills needed for cybersecurity — SSH, file navigation, permissions, encoding, and more. Each level gives you a password to unlock the next one. It sounds simple, but the later levels get genuinely tricky.

This writeup covers my approach to the first several levels, with explanations of why each command works, not just what to type.

Level 0 — Connecting via SSH

Goal: Log into the game server using SSH.

ssh -p 2220 bandit0@bandit.labs.overthewire.org

Concept: SSH (Secure Shell) creates an encrypted connection to a remote server. The -p 2220 flag specifies a non-standard port — SSH normally uses port 22, but Bandit runs on 2220. This is common in CTFs and real-world setups to avoid automated scanners.

Level 0 → 1 — Reading a file

Goal: Find the password stored in a file called readme in the home directory.

ls
cat readme

Concept: ls lists files in the current directory. cat (concatenate) outputs file contents to the terminal. These are the two most fundamental Linux commands you'll ever use. In security, reading files you shouldn't have access to is half the game.

Level 1 → 2 — Dashed filename

Goal: Read a file called - (just a dash).

cat ./-

Concept: A bare - is interpreted by most commands as "read from stdin" rather than a filename. Prefixing with ./ tells the shell "this is a file path in the current directory." This trick appears constantly in CTFs — attackers sometimes create files with special characters to confuse administrators or scripts.

Level 2 → 3 — Spaces in filenames

Goal: Read a file whose name contains spaces.

cat "spaces in this filename"

Concept: The shell splits arguments on spaces. Quoting the filename (or escaping each space with \) tells the shell to treat the entire string as one argument. Tab completion handles this automatically — a useful habit to build early.

Level 3 → 4 — Hidden files

Goal: Find a hidden file in the inhere directory.

cd inhere
ls -a
cat ...Hiding-From-You

Concept: Files starting with . are hidden in Linux. The -a flag for ls reveals them. This is how dotfiles work (.bashrc, .ssh/, .env). From a security perspective, hidden files are a common place to stash credentials, backdoors, or configuration secrets.

Level 4 → 5 — Finding the right file type

Goal: Among several files in inhere/, find the one that contains human-readable text.

cd inhere
file ./-file*

This shows each file's type. Most will be "data" (binary), but one will be "ASCII text" — that's the password.

cat ./-file07

Concept: The file command examines file contents (magic bytes) to determine the type, regardless of extension. This is crucial in forensics — file extensions can be faked, but the file command reads the actual header bytes. In incident response, you use this to identify disguised malware.

Level 5 → 6 — Finding files by properties

Goal: Find a file somewhere under inhere/ that is human-readable, 1033 bytes, and not executable.

find inhere/ -type f -size 1033c ! -executable

Concept: The find command searches recursively with filters. -type f matches regular files, -size 1033c matches exactly 1033 bytes (c = bytes), and ! -executable excludes executable files. In real security work, find is essential for hunting suspicious files — looking for recently modified files, SUID binaries, or world-writable directories.

Level 6 → 7 — Finding files across the system

Goal: Find a file owned by user bandit7, group bandit6, and exactly 33 bytes.

find / -user bandit7 -group bandit6 -size 33c 2>/dev/null

Concept: Now we search the entire filesystem (/). The 2>/dev/null redirects error messages (permission denied on directories you can't access) to nowhere, so you only see the actual result. The -user and -group flags filter by ownership. Understanding file ownership and permissions is fundamental to Linux security — privilege escalation often exploits misconfigured ownership.

Key takeaways

  • Every command is a building block. cat, ls, find, and file aren't just "basic" commands — they're the core toolkit for forensics, incident response, and penetration testing.
  • Special characters matter. Dashes, spaces, dots — the shell interprets them differently, and attackers exploit this.
  • Redirect stderr. 2>/dev/null is your best friend when searching large directory trees. Learn redirection early.
  • File metadata tells stories. Size, ownership, permissions, and type reveal more than the filename ever will.
  • CTFs build real skills. These aren't just puzzles — every technique here maps directly to real-world security work.

Bandit continues for 30+ levels, covering topics like encoding, cryptography, git secrets, cron exploitation, and privilege escalation. I'll post more writeups as I work through them.
