Building a GPA Calculator for RP Students

March 2026

The problem

Republic Polytechnic students calculate GPA differently from universities. Each module has credit units, and grades map to specific grade points. But there was no dedicated tool for RP students to quickly calculate their semester GPA or cumulative CGPA — most people were doing it manually or using generic calculators that didn't match RP's grading system.

What I built

A web-based GPA/CGPA calculator specifically designed for Republic Polytechnic. It supports all 41 diploma programmes, so students can select their course and see the correct module list with the right credit units.

Key features:

  • All 41 diplomas — select your programme and get the correct modules and credit units
  • Semester GPA — calculate your GPA for any individual semester
  • Cumulative CGPA — track your overall CGPA across multiple semesters
  • localStorage persistence — your grades are saved in the browser so you don't lose them when you close the tab
  • PDF export — generate a clean PDF report of your GPA breakdown

Technical decisions

I kept the stack deliberately simple: vanilla HTML, CSS, and JavaScript. No frameworks, no build tools, no npm packages. The entire app is a static site that can be hosted for free on GitHub Pages.

Why no framework? Because the problem doesn't need one. A GPA calculator is fundamentally a form that does arithmetic. Adding React or Vue would have introduced build complexity, increased load times, and made it harder for other students to understand and contribute to the code.

The diploma programme data (module names, credit units) is stored as a JavaScript object. This means updating it when RP changes their curriculum is as simple as editing one file.

localStorage for persistence

One of the most useful features is automatic saving. When you enter grades, they're stored in localStorage so they persist across browser sessions. This was important because students typically update their GPA calculator at the end of each semester — they shouldn't have to re-enter everything each time.

The data is stored as JSON keyed by diploma programme, so switching between programmes doesn't overwrite your data.

PDF export

Students often need to share their GPA with advisors or include it in applications. The PDF export generates a clean, formatted report showing each module, grade, credit units, and the calculated GPA/CGPA. This was implemented using the browser's built-in print functionality with a print-specific CSS stylesheet — no external PDF library needed.

What I learned

  • Build for your users. I'm an RP student, so I knew exactly what the pain points were. The best tools come from scratching your own itch.
  • Simple tech is underrated. Vanilla JS with no build step means anyone can fork, understand, and modify the code in minutes.
  • localStorage is powerful. For client-side apps that don't need a backend, it's a simple way to add persistence without any server infrastructure.
  • Ship early. The first version was rough but usable. Feedback from classmates drove every improvement after that.
