Building MyOS: A Hobby Operating System from Scratch

March 2026

Why build an OS?

I wanted to understand what actually happens when you press the power button. Not the abstract textbook version — the real thing. What loads first? How does the screen show pixels? How does the keyboard work without drivers someone else wrote? Building MyOS answered all of that and more.

MyOS is a bare-metal x86 operating system that boots via GRUB into an 800x600 graphical desktop. It has no dependencies on Linux, Windows, or any existing OS. Everything — from the memory allocator to the window manager to the TCP stack — is written from scratch in C and x86 Assembly.

Starting from nothing

The first milestone was getting a single pixel on screen. That alone took days. You need a multiboot-compliant bootloader, a framebuffer set up by GRUB, and a way to write to video memory. There's no printf, no malloc, no standard library at all. You compile with -ffreestanding -nostdlib -nostdinc and everything you use, you write yourself.

The bootloader (boot.asm) sets up the GDT, loads the IDT, installs ISR/IRQ stubs, and jumps to kernel_main in C. From there, the kernel initializes the PIT timer at 100Hz, sets up the PS/2 keyboard and mouse drivers, and enters the desktop loop.

The GUI: a Windows 7-inspired desktop

I wanted a real windowed desktop, not just a terminal. The window manager supports draggable windows with title bars, minimize/close buttons, a taskbar with a clock, and a start menu. Each application runs inside a window and receives keyboard and mouse input.

The desktop uses a simple compositing approach: draw the background, then each window back-to-front, with the active window on top. The cursor is saved/restored to avoid full redraws on every mouse move.

There are now 14 built-in apps: Notepad, Calculator, System Info, Terminal, File Manager, Breeze Browser, Paint, Snake, Minesweeper, Tetris, 2048, Flappy Bird, Solitaire, and Checkers.

The terminal: 50+ Linux-style commands

The terminal was where I spent the most time. It supports ls -l, grep, find, tree, cat, head, tail, cp, mv, pipes (cmd | grep pattern), environment variables, aliases, tab completion, command history with arrow keys, and output redirection.

There are also fun commands: cowsay, fortune, cal, and screenfetch that shows system info with an ASCII art logo. It feels surprisingly close to a real Linux shell.

Networking from Ethernet to HTTP

The networking stack was the hardest part. I wrote a driver for the AMD PCnet-PCI II network card (the one QEMU and VirtualBox emulate), then built up from raw Ethernet frames through ARP, IPv4, UDP, TCP, DNS, and HTTP.

The wget command can fetch web pages. The Breeze Browser renders HTML. And the ai command sends HTTP POST requests to Ollama running on the host, letting you chat with an LLM from inside the OS.

Kernel fundamentals

Under the hood, MyOS has x86 paging with 4KB pages and a page fault handler, a physical page bitmap allocator, a first-fit heap allocator with coalescing, preemptive round-robin multitasking driven by the PIT timer, and an ATA/IDE disk driver with a custom filesystem (MYFS) for persistent storage.

You can save the entire filesystem to disk and it reloads on next boot. Processes have PCBs and context switch via IRQ0 with 50ms time slices.

What I learned

Building an OS taught me more about computers than any course could. Some specific takeaways:

  • Everything is memory-mapped. The framebuffer, device registers, page tables — it's all just addresses.
  • Interrupts are everything. Without IRQs, the keyboard doesn't work, the timer doesn't tick, and the network card can't receive packets.
  • Debugging without tools is brutal. No debugger, no logs, no stack traces. When something crashes, you get a triple fault and the VM reboots. I learned to use QEMU's -d int flag and GDB remote debugging.
  • Networking is layers of trust. Each protocol assumes the one below it works. When it doesn't, you stare at hex dumps for hours.
  • Freestanding C is a different language. No strlen, no memcpy, no printf. You write them all yourself and appreciate them forever after.
